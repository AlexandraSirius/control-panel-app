# Control Panel App

Приложение панели управления, разработанное на Processing, предназначенное
для работы с аппаратными контроллерами (кнопки, ручки, энкодеры, слайдеры)
с поддержкой сцен, масштабируемой верстки и последовательного (Serial)
взаимодействия с устройством.

Проект построен по модульной архитектуре с разделением логики интерфейса,
элементов управления и системных компонентов.

---

## Назначение проекта

- Визуальный интерфейс панели управления
- Управление аппаратными контроллерами
- Поддержка различных режимов работы (основной, сетевой, настройки)
- Подготовка к использованию MIDI / OSC / сетевых протоколов
- Масштабируемый и расширяемый UI

---

## Архитектура проекта

Проект состоит из следующих логических уровней:

- **Ядро приложения**
- **Сцены (экраны)**
- **UI-верстка и навигация**
- **Базовый класс органов управления**
- **Конкретные элементы управления**
- **Менеджеры конфигурации и связи**

---

## Структура файлов проекта

### Основные файлы приложения

#### `sketch_00_ControlPanelApp.pde`
Главный файл приложения.
Содержит глобальные переменные, общие структуры данных и инициализацию
основных компонентов интерфейса. Является логическим ядром приложения.

---

#### `sketch_01_main.pde`
Основной Processing-скетч.
Содержит функции `setup()` и `draw()`, отвечает за запуск приложения,
рендер интерфейса и вызов логики сцен.

---

### Сцены (экраны приложения)

#### `SceneMain.pde`
Главная сцена приложения.
Отвечает за основной режим управления, отображение органов управления
и взаимодействие пользователя с контроллером.

---

#### `SceneNetwork.pde`
Сцена сетевых настроек.
Используется для отображения и редактирования сетевых параметров
(IP-адреса, порты и другие параметры подключения).

---

#### `SceneSettings.pde`
Сцена общих настроек приложения.
Содержит системные и пользовательские параметры, влияющие на поведение
приложения.

---

### Верстка и навигация интерфейса

#### `sketch_02_ui_layout.pde`
Файл компоновки интерфейса.
Отвечает за геометрию UI, размеры элементов, масштабирование и сетку
расположения элементов интерфейса.

---

#### `sketch_03_ui_right_panel.pde`
Правая панель интерфейса.
Содержит логику отображения и редактирования параметров выбранного
элемента управления.

---

#### `sketch_04_tabs.pde`
Файл вкладок интерфейса.
Реализует верхнее меню вкладок и логику переключения режимов
(например: основной режим, сеть, настройки).

---

#### `sketch_05_utils.pde`
Вспомогательные функции.
Содержит утилитарные методы, используемые в разных частях проекта
(масштабирование координат, расчеты размеров и т.п.).

---

### Менеджеры

#### `config_manager.pde`
Менеджер конфигурации.
Отвечает за хранение, обработку и подготовку параметров конфигурации
приложения (в будущем — сохранение и загрузка настроек).

---

#### `serial_manager.pde`
Менеджер последовательного порта (Serial).
Реализует взаимодействие с аппаратным устройством:
подключение, отправку команд и приём данных.

---

### Базовый класс элементов управления

#### `sketch_10_ControlBase.pde`
Базовый абстрактный класс для всех органов управления.
Содержит общие поля (позиция, размер, значение) и базовые методы
отрисовки и обработки событий.

Все элементы управления наследуются от данного класса.

---

### Элементы управления

#### `sketch_11_Knob.pde`
Класс ручки (Knob).
Реализует вращаемый орган управления с аналоговым значением.

---

#### `sketch_12_Button.pde`
Класс кнопки.
Реализует цифровой орган управления (нажатие / отпускание).

---

#### `sketch_13_BigButton.pde`
Класс большой кнопки.
Используется для визуально выделенных или функционально важных кнопок
(например, кнопка отправки команды).

---

#### `sketch_13_Screen.pde`
Класс экранного элемента.
Используется для отображения информации без прямого пользовательского ввода.

---

#### `sketch_14_Encoder.pde`
Класс энкодера.
Поддерживает вращение влево/вправо и нажатие.

---

#### `sketch_14_Slider.pde`
Класс слайдера.
Реализует линейный аналоговый орган управления.

---

#### `sketch_15_Lever.pde`
Класс рычага.
Реализует вертикальный орган управления с изменяемым значением.

---

#### `sketch_16_ScreenDummy.pde`
Заглушка экранного элемента.
Используется для тестирования и отладки интерфейса.

---

## Технологии

- Processing (Java mode)
- Модульная архитектура UI
- Serial communication (подключение аппаратных устройств)

---

## Статус проекта

Проект находится в активной разработке.
Архитектура позволяет легко добавлять новые сцены и элементы управления.
